1. Representing Text

Character Encoding: The universal dictionary mapping letters to numbers.

UTF-8

Popular
  • Variable byte length (1-4)
  • 1 byte for basic Latin (A-Z)
  • Highly efficient for Web/English
Standard for Web, Linux, macOS

UTF-16

Balanced
  • 2 or 4 byte length
  • Efficient for Asian languages
  • Middle-ground approach
Used in Windows & Java

UTF-32

Simple
  • Fixed 4-byte length
  • Predictable performance
  • Memory-intensive (bulky)
Rarely used in practice

2. Representing Video

High-definition video contains millions of pixels per frame. Streaming would be impossible without Compression.

Intra-Picture

Compresses the first frame like a high-quality JPEG photo.

Inter-Picture

Instead of re-sending the picture, it only sends what changed via motion vectors.

HEVC (High-Efficiency Video Coding)

The Solution

  • • Designed for 4K and 8K resolution
  • • Aiming for 50% bit rate reduction
  • • Supports parallel processing for multi-core CPUs

3. Representing Sound

Converting continuous waves into discrete numbers.

Step 1: Sampling

Taking snapshots of amplitude at fixed intervals.

Nyquist Theorem

To capture sound accurately, the sampling rate must be >2x the highest frequency. Humans hear up to 20kHz, so CDs use 44.1kHz.

Step 2: Quantization

Assigning numerical values to samples based on Bit Depth.

High Bit Depth

Like a ruler with mm markings. Precise measurements, low noise.

Low Bit Depth

Like a ruler with cm markings. Rounding errors cause a "hiss" (distortion).

CD Standard
44,100 Hz / 16 Bit

Essential Takeaways

Binary Translation

Every complex experience is eventually stripped down to a sequence of numbers (0 and 1).

Clever Prediction

Efficiency in media comes from "prediction"—only storing what is new or changed.

Fidelity vs. Size

The quality of digital media is a trade-off between sampling/precision and storage space.
